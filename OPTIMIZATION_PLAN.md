# ESP32 USB IP 服务器并发性能优化计划

## 当前状态分析

### 1. 并发模型现状
- **问题**: 使用 `std::shared_mutex` + `transferring_data` map 管理并发传输
  - 热点: `transferring_data_mutex` 在每次提交/完成转移时都会被锁定
  - 粒度: map 是全局的单一锁，所有端点共享
  - 并发受限: 通过 `concurrent_transfer_count` 原子计数器限制（默认32）

### 2. 零拷贝现状
- **实现**: 通过 `UsbTransferPtr` 所有权转移避免拷贝 (线程 972 行)
- **限制**: 仅在特定条件下生效（需要 USB 缓冲区直接对齐）
- **统计**: 跟踪 `zero_copy_count` 和 `total_transfer_count`
- **遗漏**: 大请求时的分片是否真正零拷贝？

### 3. 网络自适应现状
- **现状**: 硬编码批量配置（max_batch_size=32, max_batch_bytes=65536）
- **缺陷**: 无根据网络状况动态调整的机制
- **机会**: 可通过网络延迟/吞吐量反馈动态调优

### 4. UAS 协议现状
- **现状**: 仅支持 USB 设备规范的标准传输
- **机会**: ESP32 USB Host 可能不原生支持 UAS，需要软件模拟

## 优化方案

### Phase 1: 优化并发模型 (降低 50% 锁竞争)
- [ ] 替换全局 map 为无锁数据结构或分段锁
- [ ] 实现 per-endpoint 锁而非全局锁
- [ ] 改进并发限制策略（基于内存/CPU负载）

### Phase 2: 零拷贝发送完全覆盖 (目标 >90% 零拷贝率)
- [ ] 分析大请求的拷贝路径
- [ ] 优化分片逻辑以保持零拷贝特性
- [ ] 添加自动回退机制

### Phase 3: 自适应请求大小 (延迟 <20ms @ 100Mbps)
- [ ] 实时网络性能监测
- [ ] 动态批处理参数调整
- [ ] 拥塞控制算法集成

### Phase 4: UAS 协议研究与原型
- [ ] 评估 ESP32 USB Host 能力
- [ ] 设计 UAS 兼容层
- [ ] 原型实施与性能对比

## 性能指标

| 指标 | 当前 | 目标 | 方法 |
|------|------|------|------|
| 锁竞争 | 高 | 低 | 无锁/分段锁 |
| 零拷贝率 | <80% | >90% | 优化分片逻辑 |
| 平均延迟 | 未知 | <15ms | 自适应批处理 |
| USB 吞吐量 | 有限 | 最大化 | 并发优化 |
| 内存消耗 | 高 | -20% | 零拷贝覆盖 |
